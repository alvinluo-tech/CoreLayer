## Per-Phase Detail

### Phase 11: Reliability Foundations

- [x] 11.1 Provider Retry — ✓ Verified. `provider.ts` implements retry on 429/503, max 3 attempts, exponential backoff (1s, 2s). Note: roadmap says "1s, 2s, 4s" but only 2 delay intervals execute with 3 attempts. Pattern is correct.
- [x] 11.2 DeadHost Exponential Backoff — ✓ Verified. `dead-host.ts` implements 20s→40s→80s→160s cap, consecutive failure tracking, reset on success.
- [x] 11.3 ConfigManager Cache — ✓ Verified. `config-manager.ts` has mtime-based cache for `getConfig()` and `getCredentials()`, exported `invalidateConfigCache()`.
- [x] 11.4 Stream Abort — ✓ Verified. `conversation.ts` creates AbortController, passes signal to streamText, timeout from configManager, cleanup on finish.
- [x] 11.5 Client SSE Disconnect — ✓ Verified. Both `chat.ts` and `conversations.ts` detect disconnect via Hono signal and abort upstream.
- [x] 11.6 Integration & Testing — ⚠️ Tests exist for provider retry, dead-host, and config cache. **Missing**: no route-level tests for SSE disconnect in `chat.ts` or `conversations.ts`; `conversation.test.ts` does not test AbortController/stream timeout.

### Phase 12: Model Routing & Token Intelligence

- [x] 12.1 Wire ModelGateway — ✓ Fixed. `inferTaskContext` now infers `requiresPrivacy` (password/credential/personal data keywords) and `requiresVision` (image/screenshot/photo keywords) from the user message.
- [x] 12.2 Memory Relevance Threshold — ✓ Verified. `MEMORY_MIN_SCORE = 0.3` constant exported and used. **Minor**: roadmap says "configurable via config" but it's a hardcoded constant, not config-driven.
- [x] 12.3 Token Usage Tracking — ✓ Verified. `promptTokens`/`completionTokens` columns, `onStepFinish` accumulation, DB persistence via `updateTokenUsage`.
- [x] 12.4 Frontend Token Display — ⚠️ Partial. Sidebar shows token count per conversation, OverviewPage shows totals. **Gap**: no token display in the active conversation's `ChatPanel` header/footer.
- [x] 12.5 Cost Estimation — ✓ Verified. Pricing in model profiles, `estimateCost()` in settings-usage.ts, `GET /api/settings/usage` endpoint, frontend display in OverviewPage.
- [x] 12.6 Integration & Testing — ⚠️ Routing/gateway tests exist (17 tests across gateway.test.ts and router.test.ts). **Missing**: no daemon-side tests for token tracking, cost computation, or memory threshold filtering.
